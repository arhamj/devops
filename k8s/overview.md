# Overview

## Container orchestration features
- High availability
- Scalability
- Disaster recovery

## Components
1. Node
2. Pod
3. Service
4. Ingress
5. Config map
6. Secrets
7. Volume
8. Deployment
9. Stateful set

## Worker node
1. Container runtime
2. Kubelet
3. Kube proxy
   - Forward requests from service to pods

## Master node
1. API server
2. Scheduler
3. Controller manager
4. etcd

## Minikube
Single node setup for local development with the following components
- 1 Master process
- 1 Worker process
